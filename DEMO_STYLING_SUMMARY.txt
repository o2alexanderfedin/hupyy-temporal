==============================================================================
KEY TECHNICAL NOTES
==============================================================================

Framework Limitations:
  - Streamlit doesn't support external CSS files
  - All styling must be via st.markdown(..., unsafe_allow_html=True)
  - Theme config only supports limited properties
  - Component styling requires specific data-testid selectors

Opportunities:
  - Easy to inject custom CSS via markdown
  - Can create wrapper functions for consistency
  - Session state enables dynamic themes
  - Streamlit's st.html() provides more control (newer feature)

Performance Considerations:
  - Keep CSS minimal to avoid page load delays
  - Use CSS classes instead of inline styles
  - Cache styles in session state if needed
  - Test with long-running operations

Accessibility:
  - Ensure color contrast meets WCAG standards
  - Add focus states for keyboard navigation
  - Test with screen readers
  - Support reduced motion preferences
